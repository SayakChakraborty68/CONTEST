the question is saying that we can add 2 adj element (a[i-1] and a[i] for 1<=i<=n-1) if they have same parity 
means both number should be even or both number should be odd 

Now we can add 2 adj element with same parity and converge them in a single element....
and the question is asking how many different array are possible are there......
so lest do some observation :-
obs1:- 
                   a[i-1]       a[i]
        case 1:-   odd         even     addition can not perform but we can say that the last element of the aray                                                                    
                                                    always will be even element 
                                                  
        case 2:-   odd         odd         we can do add operation so there we have to choice ither to add the element 
                                                       and get the last element as even or do nothing and get last element as odd

        case 3:-   even         even     here also  we can  do addition operation and get last element as even and if we 
                                                     dont do addition then also we can get the last element as even also 

        case 4:-   even        odd     like the 1st case we cannot add these 2 elements we will get always a odd ending 
                                                   element

#### there is a question that can come to yous mind that why we always take concerns about the last element always we will discuss this further detail ..... );


so let initialize some dp states

dp_o[i] = number of different arrays can be generated till index i such that last element of the array is an odd element

dp_e[i] = number of different arrays can be generated till index i such that last element of the array is an even element

 Lest think about transition states----
lest say we have dp_o and dp_e till i-1 index
and there are 2 possible case 
Case1 :-- a[i] is an even element (a[i]&1 ==0)
               and there we have 2 choices 
               Choice1:- do nothing 
                               so a[i] will concatenate with all arrays ending with odd element(dp_o[i-1]) and all arrays ending 
                               with even element (dp_e[i-1]) and then new arrays will be all element ending with **even**  
                               element
                                so the transition state will be 
                                    dp_e[i] = dp_e[i-1] + dp_o[i-1];
                                    dp_o[i] = 0 ;
             Choice2:- do addition 
                           we can do addition with the last elements of all the arrays with**even**ending elemnts(dp_o[i-1])
                            and the ending elements of those arrays will be even
                             dp_e[i]  += dp_e[i-1];
                             dp_o[i] = 0 ; (as addition always give even ending element)

so if a[i] is even then
                             dp_o[i] = 0;
                             dp_e[i] = dp_o[i-1] + 2*dp_e[i-1];

Case2 :-- a[i] is an odd element (a[i]&1 ==1)
               and there we have 2 choices 
               Choice1:- do nothing 
                               so a[i] will concatenate with all arrays ending with odd element(dp_o[i-1]) and all arrays ending with even element (dp_e[i-1]) and then new arrays will be all element ending with odd element
 so the transition state will be 
                                    dp_o[i] = dp_e[i-1] + dp_o[i-1];
                                     dp_e[i] = 0;
             Choice2:- do addition 
                            we can do addition with the last elements of all the arrays with odd ending elements(dp_o[i-1])
                            and the ending elements of those arrays will be even 
                             dp_e[i]  = dp_o[i];
                              dp_o[i] will remain same
                     There is a special case if a[i] and a[i-1] both are odd then we can do addition which will give even element but we will miss all the cases before index i-2 which will give us also ending element as even
                                       dp_e[i] += dp_e[i-2] if(i>1)

#include <bits/stdc++.h>
using namespace std;

int main() {
    int t;
    cin >> t;
    while(t--){
        int n;
        cin >> n;
        vector<int> a(n);
        long long MOD = 998244353;
        for(int i=0;i<n;i++) cin >> a[i];
        if(n==1){
            cout << 1 << endl;
            continue;
        }
        vector<long long> dp_o(n,0),dp_e(n,0);
        if(a[0]&1){
            dp_o[0] = 1;
            dp_e[0] = 0;
        }
        else{
            dp_o[0] = 0;
            dp_e[0] = 1;
        }
        for(int i=1;i<n;i++){
            if(a[i] & 1){
                dp_o[i] = (dp_o[i-1]+dp_e[i-1])%MOD;
                dp_e[i] = dp_o[i-1];
                if(i > 1 && a[i-1]&1) dp_e[i] =  (dp_e[i]+dp_e[i-2])%MOD;
            }
            else {
                dp_o[i] = 0;
                dp_e[i] = (dp_o[i-1]+(dp_e[i-1]*2)%MOD)%MOD;
            }
        }
        cout << (dp_o[n-1]+dp_e[n-1])%MOD << endl;
    }

}
